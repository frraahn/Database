# 🗄️ Database Design 
Welcome to my repository for the **Database (WBL)** course at Universiti Teknologi Malaysia (UTM). This space serves as a portfolio documenting our team-based relational database design lifecycle alongside my individual practical database laboratory scripts.

---

## 📂 Repository Structure & Portfolio Breakdown

### 🔹 Project: Koperasi KADA Automated System Database (Group Team: SYNAPSE)
* **General Summary:** Operating as a backend design team, we developed a complete relational database management subsystem for the Koperasi KADA Automated System. Following a structured SDLC approach, we extracted stakeholder business rules to build a global conceptual layout, translated it into a normalized logical database blueprint, mapped out explicit operational workflows with activity diagrams, and established transactional requirements to safely manage cooperative memberships, loans, and automated reports.
* **Core Deliverables:** * Conceptual Database Design Report (`/SECP2523-P2_ConceptualDesignReport Synapse (1).pdf`)
  * Logical Database Design Report (`/SECP2523-P3_Synapse.pdf`)

### 🔹 Lab Work: SQL Engineering & EERD Modeling
* **General Summary:** A collection of individual practical assessments focusing on relational database architecture, schema modification, and data manipulation. The work spans designing advanced schematic systems such as creating Enhanced Entity Relationship Diagrams (EERD) to writing native, optimized SQL queries to handle production-level profile overrides, automated constraints, and active module creation.
* **Core Deliverables:** * EERD Conceptual Architecture Mapping (`/SECP2523 DATABASE ASSIGNMENT 2.pdf`)
  * SQL Scripting Library - DDL & DML (`/LAB ASSIGNMENT DDL & DML.pdf`)

## 💡 Course Reflection & Key Takeaways
This course brought relational database management systems to life by showing me how to turn unstructured client requirements into organized, highly performant tables. Building the backend schema for the Koperasi KADA system taught me how to scale a database correctly using conceptual ERDs to map relationships, logical dictionaries to verify strict constraints, and normalization practices to prevent data redundancy. Additionally, writing the hands-on DDL and DML scripts showed me how structural engineering translates directly into actual system execution via optimized SQL statements. Ultimately, this mix of conceptual group planning and strict relational script writing has given me the confidence to model, query, and optimize secure data architectures.
